## Reconnaissance & Target Discovery

### Passive Recon

1. Gather public info (DNS, WHOIS, certificates, company structure)
2. Search public data leaks (Google Dorks, GitHub, Pastebin, etc.)

### Active Recon

1. Discover all active hosts on the target network/IP range/subnet(s). Document all active hosts to Obsidian notes
- [ ] NMAP Host Discovery Scan
- [ ] ICMP sweep (ping or fping)
- [ ] TCP/UDP host discovery (`nmap -sn`, masscan)
- [ ] ARP scanning (if on same subnet)
- [ ] Add discovered hostnames to `/etc/hosts` file

2. For each active host, scan ALL TCP/UDP ports. Document each open port in Obsidian.
- [ ] NMAP TCP port scanning

```bash
nmap -A --open $rhost -oA nmap/tcp
```

- [ ] NMAP UDP port scanning

```bash
nmap -sU --open $rhost -oA nmap/udp
```

3. For each open port, run service version scan with scripts and OS detection.
- [ ] NMAP service enumeration and OS detection

```bash
nmap -sC -sV -O --open -p <ports> $rhost -oA nmap/detailed
```

- [ ] Netcat banner grabbing (manual confirmation)
- [ ] Document services and versions in Obsidian

4. For each detected service, do individual service enumeration to look for more information and vulnerabilities

5. Check for vulnerabilities in discovered services/versions
- [ ] Search Metasploit for service exploits
- [ ] Search ExploitDB for service exploits
- [ ] Look at NMAP script output for vulnerabilities or misconfigurations (e.g. anonymous login)
- [ ] Look for OS version exploits
- [ ] Search Google for `<Service> <Version> Exploit GitHub`
- [ ] Document discovered vulnerabilities in Obsidian

6. Check file share services (FTP, SMB, etc.) for anonymous logon and credential files
- [ ] FTP Enumeration [[21-ftp]]
- [ ] SMB Enumeration [[139-445-smb-netbios]]

7. Check for write access over a share (SMB, FTP)
- [ ] Capturing Hashes with Malicious `.lnk` File ([[3.1.AD-Exploitation]] in NTLM Theft part)
- [ ] Capture hashes with SCF on a File Share (no longer works on Windows Server 2019+) ([[3.1.AD-Exploitation]] in NTLM Theft part)
- [ ] For FTP if we have write access we can write the .htaccess and shell then access into the webserver. [[21-ftp]]

4. If webserver(s) exist, see [[2.Web-Enumeration-Methodology]]
