### Default Methodology

1. Run Windows Priv Esc Automation Scripts. Save output and transfer to attack box. 
- [ ] winPEAS
- [ ] Seatbelt
- [ ] PowerUp / SharpUp
- [ ] JAWS (PowerShell-based; useful if custom executables are blocked)
- [ ] SessionGopher
- [ ] Bloodhound

2. Perform basic box enumeration once a foothold is established
- [ ] Gathering Network Information
- [ ] Gathering System Information
- [ ] Process Enumeration
- [ ] User and Group Enumeration

1. Look at access rights of user (user privileges)
```Bash
whoami /priv
whoami /all
```
- [ ] SeImpersonate / SeAssignPrimaryToken → JuicyPotato / RoguePotato / PrintSpoofer
- [ ] SeDebugPrivilege
- [ ] SeTakeOwnershipPrivilege

4. Check if user is a member of privileged groups    
```
whoami /groups
```
- [ ] Backup Operators
- [ ] Event Log Readers
- [ ] DnsAdmins
- [ ] Hyper-V Admins
- [ ] Print Operators
- [ ] Server Operators
- [ ] WSUS Administrators

5. Check for Weak File/Service Permissions
- [ ] Permissive File System ACLs
- [ ] Weak Service Permissions
- [ ] Unquoted Service Path
- [ ] Permissive Registry ACLs
- [ ] Modifiable Registry Autorun Binary

5. Check for saved credentials
```Bash
cmdkey /list
```
- [ ] If we get access this way, run mimikatz to try to extract plaintext password

6. Look for services running on internal ports not accessible from outside
```Bash
netstat -ano
```
- [ ] Databases we can connect to?
- [ ] Vulnerable Services

7. Check for additional NICs
```Bash
ipconfig
```

8. Look for vulnerable applications and services
```Bash
wmic product get name
```

9. Look for interesting files on the server that may have credentials or sensitive info
- [ ] Credential Hunting
- [ ] Dumping Hashes / Credentials (Mimikatz)

10. Pillage for credentials or other interesting information
- [ ] Pillaging Applications
- [ ] Accessing Instant Messaging Clients Through Cookies
- [ ] Pillaging the Clipboard + Keylogging
- [ ] Roles and Services (pillaging backups)

11. Look for scheduled tasks that we can modify

12. Look for credentials in process command line

13. Check for 'Always Install Elevated' setting and exploit with MSI package

14. Capture hashes with a Malicious LNK file or SCF file
- [ ] **ESPECIALLY USEFUL IF USERS WILL VISIT A SHARE WE HAVE UPLOADED TO**

15. Look for Kernel / OS exploits
- [ ] Zero Logon
- [ ] PrintNightmare
- [ ] EOL Systems (Windows Server 2008, Windows 7, etc.)

16. Attempt to bypass UAC Controls if present

17. DLL Injection

18. Common Vulnerabilities and Vulnerable Programs
- [ ] Docker Desktop Community Edition before 2.1.0.1 (CVE-2019-15752)
- [ ] Windows Certificate Dialog (CVE-2019-1388)

19. Attempt to Capture Network Traffic with Inveigh/Responder, Wireshark, or Snaffler
- [ ] LLMNR/NBT-NS Poisoning From Windows/Linux

20. Enumerate User/Computer Description Fields for cleartext credentials

21. If the system has `.vhd`, `.vhdx`, `.vmdk` files, mount them to dump machine hashes

22. Check for Active Directory Certificate Services (AD CS) attacks

23. If trying to be evasive or custom executables are locked down, check LOLBAS
