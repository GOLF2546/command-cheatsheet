### Initial (Uncredentialed) Enumeration

#### Host Identification

1. Create a document for all discovered hosts (hostnames and IP addresses)
2. Start Wireshark and listen for Layer 2 (ARP, MDNS, NBNS) traffic to discover IP addresses and hostnames
3. Start Responder in Analyze mode to discover IP addresses and hostnames
4. Perform an Fping ICMP Sweep to find all hosts that respond to ICMP echo request
5. Perform an NMAP identification scan in case it picks up anything missed
6. With all discovered hosts, perform NMAP scan to determine running services
- [ ] Save all ports/services for each discovered host; pay special attention to DCs, RDP, naming conventions
- [ ] Look for any quick wins (outdated software or OS)

#### User Identification

1. Create a document for all discovered users
2. Attempt to abuse SMB Null Session against the DC with rpcclient or enum4linux to grab all users
3. Attempt anonymous LDAP search against the DC with ldapsearch or windapsearch
4. Try Impacket's `lookupsid.py` for SID/RID brute forcing
5. Use Kerbrute and common wordlists to brute force usernames against a discovered DC
6. Check for users that do not require Kerberos pre-auth (ASREPRoasting)
7. Look for systems that can be exploited to gain SYSTEM level access (essentially like getting AD credentials)

#### User Foothold

1. Start Responder/Inveigh on network interface to listen for NTLM users and hashes. Attempt to crack with Hashcat/John
2. Attempt a password spray on users identified during user identification
- [ ] Attempt to gather the password policy of the organization first
- [ ] Password spray using common passwords like: `Welcome1`, `ChangeMe1`, `SeasonYear (Spring2025)`, username as the password, etc.

### Credentialed Enumeration / Exploitation

#### Host Identification

1. Run Bloodhound with discovered credentials (mark controlled assets as owned)
2. Use ldapdomaindump to identify all domain joined computers
3. Enumerate accessible shares on servers with NetExec, SMBMap, PowerView, or Snaffler

#### User Identification

1. Run Bloodhound with discovered credentials
2. Gather the domain password policy using the discovered credentials
3. Use NetExec to get all users, groups, and logged-on users against the server you have credentials for
4. Gather a list of Domain Admins or Privileged users using:
- [ ] Windapsearch
- [ ] PowerView
- [ ] Bloodhound
- [ ] AD PowerShell module
5. Credentialed Enumeration from Linux
6. Credentialed Enumeration from Windows

#### Foothold Enumeration

1. Run Bloodhound with discovered credentials
2. Enumerate security controls in place (Defender, AppLocker, PowerShell Constraints, LAPS)
3. Look for other logged-on users with CME to see if we can dump credentials with Mimikatz or Rubeus
4. Look for kerberoastable accounts through Bloodhound, PowerView, GetUserSPNs.py
5. Look at owned users for abusable ACL entries (ForceChangePassword, AddMember, GenericAll, etc.) - easiest via BloodHound
6. Check Bloodhound for CanRDP, CanPSRemote, or SQLAdmin abilities to move laterally

#### Pivoting

1. Run chisel for windows on the Windows pivot host
- [ ] Connect with the linux chisel client on the attack host
- [ ] Modify `proxychains.conf` to match the established proxy
- [ ] Run commands with proxychains
2. Check Bloodhound for CanRDP, CanPSRemote, or SQLAdmin abilities to move laterally

### Exploitation

1. Look for kerberoastable accounts through Bloodhound, PowerView, GetUserSPNs.py

2. Grab all TGS tickets with GetUserSPNs.py and save to a file. Attempt to crack with Hashcat (GPU rig preferred)

3. Abuse over-permissive ACL entries to gain control of more users and move laterally:
- [ ] `ForceChangePassword` → change a user's password
- [ ] `AddMember` → add a member we control to a privileged group
- [ ] `GenericAll/GenericWrite` → create an SPN for account and kerberoast their password hash
- [ ] `GenericAll` → change the user's password
- [ ] `DS-Replication-Get-Changes-All` → DCSync attack
- [ ] `AddKeyCredentialLink` → get user's NTLM Hash
- [ ] ACL Abuse from Linux

4. Check Bloodhound for CanRDP, CanPSRemote, or SQLAdmin abilities. Abuse these rights and look for sensitive info on new machines

5. Check for common vulnerabilities to escalate privileges or move laterally:
- [ ] NoPac
- [ ] PrintNightmare
- [ ] PetitPotam

6. Check for common misconfigurations:
- [ ] Exchange group permissions
- [ ] MS-RPRN Printer bug
- [ ] MS14-068
- [ ] Sniff for LDAP credentials
- [ ] Enumerate DNS records for interesting servers
- [ ] Look for user passwords and notes in AD user descriptions
- [ ] Check for `PASSWD_NOTREQD` field on users and test for weak/no passwords
- [ ] Look for credentials and interesting files on SMB shares (manually or with Snaffler)
- [ ] Check for `DONT_REQ_PREAUTH` field and ASREPRoast any discovered users
- [ ] Check for GPOs with write access (check via BloodHound)
- [ ] Resource Based Constrained Delegation, Constrained Delegation, Unconstrained Delegation
- [ ] Active Directory Certificate Services Attacks

7. Check for Group Policy Preferences (GPP) Passwords
8. Check for Active Directory Certificate Services (AD CS) attacks

### Attacking AD Trusts (Parent Domain)

1. Discover any current domain trusts with `Get-ADTrust`, `Get-DomainTrust` (PowerView), or Bloodhound
2. From a machine with Domain Admin privileges, attempt an ExtraSIDs attack to create an Enterprise Admin user in the parent domain
3. Child -> Parent Attacks - Windows
4. Child -> Parent Attacks - Linux

### Attacking AD Trusts (Cross Forest)

1. Discover any current domain trusts with `Get-ADTrust`, `Get-DomainTrust` (PowerView), or Bloodhound
2. Attempt cross-forest kerberoasting
3. If admin accounts share names across domains and one is compromised, try reused credentials
4. Check for SIDHistory abuse
5. Cross-Forest Trust Abuse - Windows
